# Governance Workflows - Check Condition

This node evaluates conditions on data assets.

Examples:

- Is a description provided?

<img src="/public/images/how-to-guides/governance/CheckConditionDescription.png" alt="Check Condition Node" />

- Are owners assigned?

<img src="/public/images/how-to-guides/governance/CheckConditionOwners.png" alt="Check Condition Node" />
<img src="/public/images/how-to-guides/governance/CheckOwnerAaronJohnson.png" alt="Check Condition Node" />

- Is the asset Tier 1?

<img src="/public/images/how-to-guides/governance/CheckConditionTier1.png" alt="Check Condition Node" />

You can define multiple conditions.

<img src="/public/images/how-to-guides/governance/check1.png" alt="Check Condition Node" />

<img src="/public/images/how-to-guides/governance/check2.png" alt="Check Condition Node" />

Each Check Condition node has two output branches:

- **TRUE** – When the condition is satisfied.
- **FALSE** – When the condition fails.

    Both branches must be connected for a complete workflow path.

<img src="/public/images/how-to-guides/governance/check3.png" alt="Check Condition Node" />
