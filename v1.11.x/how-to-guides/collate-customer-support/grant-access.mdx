# How to Grant Temporary Access to Collate Support Team Members

This guide outlines the steps required to give temporary support access in the Collate platform for the support team.

## Steps to Grant Access

### 1. Navigate to the Support Application

- Go to **Settings → Applications → Support**

<img src="/public/images/how-to-guides/support/support-1.png" alt="Settings → Applications → Support" />

### 2. Add the Collate Support Access

- In the **Add Support** section:
  - Enter the support email, for example `ayush@getcollate.io`
  - Set **Access Role** to `view only` or `Admin`.
  - Set the **Access Duration** to `1 hour`, `2 hours`, or `4 hours`

<img src="/public/images/how-to-guides/support/support-2.png" alt="Add the Collate Support Access" />

### 3. Grant Access

- Click the **Grant Access** button.

<img src="/public/images/how-to-guides/support/support-3.png" alt="Grant Access" />

- Access will be granted to the specified support email address with the configured role and duration. This can be verified in the access logs.

<img src="/public/images/how-to-guides/support/support-4.png" alt="Access Logs" />

- In the configuration section, you can select the delivery method as Email, Slack, or both.

<img src="/public/images/how-to-guides/support/support-5.png" alt="Configuration" />

<Note>
If your organization uses Single Sign-On (SSO), you must **disable the Enforce Principal Domain** to allow access for emails from other domains (e.g., `gmail.com`). Navigate to **Settings → SSO → Configure** to disable the **Enforce Principal Domain** setting.
<img src="/public/images/how-to-guides/support/support-6.png" alt="Configuration" />
</Note>
